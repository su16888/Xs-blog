## 🚀 快速部署（5步完成）

### 第 1 步：获取源码

```bash
# 克隆仓库
git clone https://github.com/su16888/Xs-blog.git
cd Xs-blog

# 或下载解压后进入目录
cd Xs-blog
```

### 第 2 步：初始化数据库

#### MySQL 方式 (推荐)

```bash
# 创建数据库
mysql -u root -p -e "CREATE DATABASE xsblog888 CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 导入数据库结构
mysql -u root -p xsblog888 < database/install.sql
```

#### SQLite 方式 (轻量级)

```bash
# 创建数据库目录
mkdir -p backend/database

# 初始化数据库
sqlite3 backend/database/xsblog.db < database/install-sqlite.sql
```

#### PostgreSQL 方式 (企业级)

```bash
# 连接到 PostgreSQL
psql -U postgres

# 创建数据库
CREATE DATABASE xsblog WITH ENCODING 'UTF8';

# 创建用户并授权
CREATE USER xsblog_user WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE xsblog TO xsblog_user;

# 退出
\q

# 导入数据库结构
psql -U xsblog_user -d xsblog -f database/install.sql
```

✅ **完成后**：数据库已创建，包含默认管理员账号（admin/admin123）

---

### 第 3 步：配置后端

```bash
cd backend

# 安装依赖
npm install

# 复制配置文件
cp .env.example .env
```

**编辑配置文件：**

Windows 用户：
```cmd
notepad .env
```

Linux/Mac 用户：
```bash
nano .env
```

**必须修改的配置项：**

```bash
# 数据库配置 (MySQL)
DB_HOST=localhost
DB_PORT=3306
DB_NAME=xsblog888
DB_USER=root
DB_PASSWORD=你的MySQL密码

# 数据库配置 (SQLite，三选一)
DB_TYPE=sqlite
DB_PATH=./database/xsblog.db

# 数据库配置 (PostgreSQL，三选一)
DB_TYPE=postgres
DB_HOST=localhost
DB_PORT=5432
DB_NAME=xsblog
DB_USER=xsblog_user
DB_PASSWORD=your_PostgreSQL_password

# 应用配置
PORT=3001
NODE_ENV=production

# JWT 密钥 (必须修改，至少32位)
JWT_SECRET=请运行命令生成随机密钥

# CORS 配置 (前后端同服务器留空)
CORS_ORIGIN=

# 文件上传路径
UPLOAD_PATH=./uploads

# 授权配置（必填）
AUTH_CODE=向作者免费索要授权码
```

**生成随机密钥**：

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

✅ **完成后**：后端配置完成

---

### 第 4 步：配置前端

```bash
cd ../frontend

# 安装依赖
npm install
```
**配置环境变量** (`frontend/.env.production`)：
```bash
# 端口配置
PORT=3000
```

**API 地址配置（重要）**
修改文件：`frontend/public/config.js`

根据您的部署方式选择配置：

**【方式一】Nginx 反向代理模式（推荐生产环境）**
```javascript
API_URL: '/api',
```

适用场景：前端和后端通过 Nginx 统一入口访问（隐藏端口）

Nginx 配置示例：
```nginx
server {
  listen 80;
  server_name yourdomain.com;

  location / {
    proxy_pass http://127.0.0.1:3000;
  }

  location /api {
    proxy_pass http://127.0.0.1:3001;
  }
}
```

**【方式二】前后端分离模式（跨域/跨服务器）**
```javascript
API_URL: 'http://192.168.1.100:3001/api',
// 或
API_URL: 'https://api.yourdomain.com/api',
```

适用场景：前端和后端部署在不同服务器或不同域名
注意：需要在后端 .env 文件中配置 CORS_ORIGIN 允许前端域名

**【方式三】同服务器直连模式**
```javascript
API_URL: 'http://localhost:3001/api',
// 或
API_URL: 'http://你的服务器IP:3001/api',
```

适用场景：前后端部署在同一服务器，通过不同端口访问

**【方式四】自动检测模式（推荐新手使用）**
```javascript
API_URL: '',
BACKEND_PORT: '3001',
```

适用场景：前后端部署在同一服务器，端口不同
工作原理：自动使用当前访问的域名/IP + 后端端口

例如：
- 访问 http://123.4.5.6:3000 → 自动调用 http://123.4.5.6:3001/api
- 访问 http://example.com:3000 → 自动调用 http://example.com:3001/api

> ⚠️ **注意**：修改 config.js 后不需要重新构建，刷新浏览器即可生效！

✅ **完成后**：前端配置完成

**ico图标替换** `frontend/public/favicon.ico`

## ⚙️ 配置修改指南

### 场景一：前后端同服务器部署（推荐）

#### 修改后端端口

**步骤 1**: 编辑 `backend/.env`，修改 `PORT`：

```bash
PORT=3001
```

**步骤 2**: 同时修改 `backend/.env` 中的 `CORS_ORIGIN`（如果之前设置过）：

```bash
# 保持为空即可，系统会自动检测新端口
CORS_ORIGIN=
```

**步骤 3**: 重启服务：
```bash
pm2 restart xs-blog-backend
pm2 restart xs-blog-frontend
```

✅ **无需修改前端配置**，系统会自动适配新端口！

---

### 场景二：前后端分离部署

#### 修改后端 API 地址

**步骤 1**: 编辑 `backend/.env`，配置允许的前端地址：

```bash
# 单个前端地址
CORS_ORIGIN=https://frontend.yourdomain.com

# 或多个前端地址（用逗号分隔）
CORS_ORIGIN=https://frontend.yourdomain.com,https://www.yourdomain.com
```

**步骤 2**: 编辑 `frontend/.env.production`，配置后端地址：

```bash
# 方式一：完整 URL（推荐）
NEXT_PUBLIC_API_URL=https://backend.yourdomain.com/api

# 方式二：拆分配置
NEXT_PUBLIC_API_HOST=backend.yourdomain.com
NEXT_PUBLIC_API_PORT=3001
NEXT_PUBLIC_API_PROTOCOL=https
```

**步骤 3**: 重启服务：
```bash
pm2 restart xs-blog-backend
pm2 restart xs-blog-frontend
```


---

### 配置优先级说明

#### 后端 CORS 配置优先级：
1. ✅ **设置了 `CORS_ORIGIN`**：使用指定的地址
2. ✅ **未设置 `CORS_ORIGIN`**：自动允许同域名请求

#### 前端 API 配置优先级：
1. ✅ **设置了 `NEXT_PUBLIC_API_URL`**：使用指定的完整 URL
2. ✅ **设置了拆分配置**：使用 `NEXT_PUBLIC_API_HOST` + `PORT` + `PROTOCOL`
3. ✅ **未设置任何配置**：自动使用当前域名 + 3001 端口

## 🔧 常见问题

### 1. 端口被占用

**错误提示**：`EADDRINUSE: address already in use`

**解决方法**：

Windows：
```cmd
netstat -ano | findstr :3001
taskkill /PID <进程ID> /F
```

Linux/Mac：
```bash
lsof -i :3001
kill -9 <进程ID>
```

或修改端口（见上方配置修改指南）

---

### 2. 数据库连接失败

**错误提示**：`connect ECONNREFUSED`

**检查清单**：
- ✅ MySQL 服务是否运行
- ✅ `.env` 中数据库配置是否正确
- ✅ 数据库用户名密码是否正确
- ✅ 数据库 `xsblog888` 是否已创建

**测试连接**：
```bash
mysql -u root -p -e "SHOW DATABASES;"
```

---

### 3. 前端无法访问后端

**检查清单**：
- ✅ 后端服务是否已启动
- ✅ 防火墙是否阻止了端口
- ✅ 服务器安全组是否开放端口

**前后端同服务器部署**：
- ✅ 检查 `frontend/.env.production` 是否为空配置（应该为空，系统自动检测）
- ✅ 检查后端 `CORS_ORIGIN` 是否为空（应该为空）

**前后端分离部署**：
- ✅ 检查 `frontend/.env.production` 中 `NEXT_PUBLIC_API_URL` 是否正确
- ✅ 检查 `backend/.env` 中 `CORS_ORIGIN` 是否包含前端地址

pm2 delete xs-blog-backend

## 🚀 Nginx 反向代理

```nginx
server {
    listen 80;
    server_name yourdomain.com;

    # 前端代理
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }

    # 后端 API 代理
    location /api {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```
